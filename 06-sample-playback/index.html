What's happening?

The Tone.Player loads and plays audio files. This is different from synthesizing sound with oscillators - instead, we're playing back pre-recorded audio.

Key Features:

  • Loading: Audio files are loaded into memory as buffers
  • Playback: Can play, pause, stop, and seek through the audio
  • Looping: Audio can repeat continuously
  • Playback Rate: Speed up or slow down (also changes pitch!)
  • Volume: Control output level in decibels or percentage

Use Cases:

  • Playing drum loops and musical phrases
  • Triggering sound effects (footsteps, UI sounds, etc.)
  • Background music playback
  • Sample-based instruments (like a sampler or drum machine)

Key Concepts:

  • Tone.Player loads and plays audio files
  • .start() begins playback from current position
  • .stop() stops playback
  • .seek(time) jumps to a specific time in the audio
  • .loop = true enables continuous looping
  • .playbackRate changes speed (and pitch) of playback
  • .buffer contains the loaded audio data

Basic Code Example:

// Load and play an audio file
const player = new Tone.Player({
    url: "../resources/audio-samples/drum-loop-120.wav",
    loop: true,           // Enable looping
    autostart: false      // Don't start immediately
}).toDestination();

// Wait for the sample to load
await Tone.loaded();

// Play the sample
player.start();

// Control playback
player.stop();                    // Stop playback
player.seek(2);                   // Jump to 2 seconds
player.playbackRate = 1.5;        // Play at 1.5x speed
player.loop = true;               // Enable looping
