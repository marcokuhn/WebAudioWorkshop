What's happening?

HTML5 provides a simple <audio> element that can play audio files. It's great for basic playback, but has limitations:

  • Can't manipulate the audio signal (no filters, effects, etc.)
  • Limited control over timing and synchronization
  • Can't connect multiple audio sources together
  • No real-time audio synthesis
  • No access to audio data for visualization or analysis

This is why we need the Web Audio API! It gives us the power to generate, process, and analyze audio in real-time.

Key Concepts:

  • <audio> element provides basic audio playback
  • JavaScript can control playback with .play(), .pause()
  • Properties like .volume and .playbackRate offer some control
  • Good for simple use cases, but limited for audio applications
